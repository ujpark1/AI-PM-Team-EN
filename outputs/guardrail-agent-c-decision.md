# Guardrail 최종 의사결정 문서

**Agent C (프로덕트 디렉터) | 2026-02-22**

---

## 1. 배경

바이브코딩은 2025년 하반기부터 폭발적으로 성장 중. Claude Code, Cursor 등이 비개발자의 앱 개발 진입장벽을 사실상 제거했다. 그런데 이들 중 보안을 신경 쓰는 사람은 거의 없다.

이것은 시한폭탄이다. Guardrail은 바이브코딩 골드러시에서 **보안이라는 필수 곡괭이**를 파는 사업.

**이 의사결정이 중요한 이유**: 타이밍이 지금이다. 6개월 후에는 기회가 사라질 수 있다.

---

## 2. 종합 평가

### 전략적 적합성 — ★★★★★ (5/5)

- 바이브코딩 트렌드와 완벽 정렬
- "Alex" 페르소나가 실존하는 수만 명을 대표
- 경쟁 포지셔닝맵 좌상단(비개발자+단순/저렴)이 완전히 비어있음
- 기존 플레이어가 이 시장 공략하려면 제품 철학을 뜯어고쳐야 함 → **12-18개월 선점 기회**

### 시장 매력도 — ★★★★☆ (4/5)

- TAM $10B+, 바이브코더 50만-200만 명 (연 3-5배 성장)
- $19/월은 Vercel($20), Supabase($25)와 같은 가격대
- **-1점**: "바이브코더가 보안에 돈을 쓸 것인가"는 아직 검증 안 된 가설

### 실행 가능성 — ★★★★☆ (4/5)

- 기술적으로 MVP 구축 가능 (오픈소스 기반)
- 초기 고객 확보 경로 명확 (PH, Indie Hackers, X)
- **-1점**: 보안 제품의 정확도/신뢰성 확보가 쉽지 않고 PRD에서 깊이 부족

### 리스크 대비 보상 — ★★★★☆ (4/5)

**비대칭적 베팅(Asymmetric Bet)**: 초기 투자 낮음, 실패 시 손실 제한적, 성공 시 업사이드 매우 큼.

| 시나리오 | 내용 |
|---------|------|
| Bull | MRR $30K+, 카테고리 대명사, 전략적 인수 가능 |
| Base | PRD 수치 달성, 니치 안정 포지셔닝 |
| Bear | WTP 실패, 무료만 사용, 플랫폼이 기능 내장 |

## 3. 결정: GO — 조건부 승인

**Type 2 결정** (되돌릴 수 있고, 빠르게 검증 가능, 실패 비용 낮음). 현재 확신도 **75-80%**.

**GO 근거:**
1. **시장 공백이 실재** — 비개발자 친화적 보안 서비스가 지구상에 없음
2. **타이밍이 최적** — 보안 사고 터지기 전에 포지셔닝 완료해야 함
3. **투자 대비 보상이 비대칭적** — 실패해도 잃는 것 적고, 성공하면 카테고리 정의

**Kill 스위치:**

| 조건 | 기한 | 기준 |
|------|------|------|
| WTP 검증 | 런칭 후 60일 | 유료 전환 의향 15% 미만 → 피벗 |
| 무료 사용자 확보 | 런칭 후 45일 | 100명 미만 → 채널/메시지 전면 재검토 |
| 스캔 품질 | MVP 완성 시 | 탐지율 80% 미만 → 런칭 연기 |

## 4. GTM 전략

### 추천 모션: PLG (Product-Led Growth) 순수 PLG

바이브코더는 스스로 도구를 찾아 쓰는 사람들. 영업팀이 콜드콜해서 팔 수 있는 제품 아님.

**PLG 플라이휠:**
```
무료 스캔 결과 공유 (배지/스크린샷)
         ↓
새 바이브코더 유입 → 무료 스캔
         ↓
"모니터링도 시작할까요?" → 유료 전환
         ↓
지속 가치 → 리텐션
         ↓
"Secured by Guardrail" 배지 → 바이럴 루프
```

### 핵심 성장 엔진: Sticky + Viral (Hybrid)

**Sticky**: 매일/매주 보안 리포트 → 습관 형성 → 이탈 방지
**Viral**: "보안 등급 A+" 공유 카드 + "Secured by Guardrail" 배지 → 바이럴 계수 0.3+ 목표

### PRD 로드맵 피드백

| Phase | PRD 제안 | 디렉터 의견 |
|-------|---------|------------|
| Phase 1 | MVP, 무료 500명 | MCP 통합을 Phase 2로 당기기 권장 (핵심 차별화) |
| Phase 2 | 유료 50명, MRR $1,500 | 전환율 3% 미만 시 가격/기능/포지셔닝 변경 |
| Phase 3 | MCP+Vercel, MRR $6,000 | 동의 |
| Phase 4 | MRR $15,000 | 팀 확장(2-3인) 계획 필요 |

## 5. PRD 피드백

### 가장 잘 된 부분
1. **"Alex" 페르소나의 생생함** — 팀 전체가 자문할 수 있는 수준
2. **"쉬운 언어" 원칙의 일관성** — 제품 철학의 핵심 차별화
3. **경쟁 포지셔닝 분석** — 비어있는 사분면 정확히 식별
4. **"골드러시의 리바이스" 비유** — 30초 설명 가능한 강력한 내러티브
5. **Freemium 가격 구조** — 바이브코더 지불 심리와 정확히 맞음

### 보완 필요
1. **보안 스캔 기술적 깊이** — 스캔 범위, 정확도 목표 미정의
2. **수요 교육 전략** — "보안? 나중에" 인식 극복 방법 없음
3. **False Negative 대응** — "안전합니다" 후 사고 시 법적/신뢰 리스크
4. **이탈 분석** — 왜 이탈하는가에 대한 가설/대응 부재
5. **팀 구성/역량** — 보안 전문가 확보 계획 없음

### 추가 필요 내용
1. 법적/컴플라이언스 (SLA, 책임 범위)
2. 데이터 처리 정책 (코드 저장 여부)
3. 경쟁사 대응 시나리오
4. 국제화(i18n) 타임라인
5. 커뮤니티 전략
