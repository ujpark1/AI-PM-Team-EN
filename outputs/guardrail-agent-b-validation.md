## 1. 시장 경쟁력 분석

### 경쟁 강도: 분산 시장 내 미개척 니치

보안 스캔/모니터링 시장 자체는 **분산 시장**(다수 플레이어)이지만, **"비개발자 바이브코더" 세그먼트는 사실상 미개척**.

**함정 경고:** "미개척"은 두 가지 의미:
- 긍정적: 선점 기회
- **부정적: 시장 자체가 존재하지 않을 수 있다**

바이브코더가 보안에 돈을 쓸 의사가 있는지 아직 검증되지 않았다.

### 진입 장벽: 양방향 모두 낮음 (경고)

**경쟁자의 모방 장벽이 매우 낮다. 이것이 가장 큰 문제.**

| 모방 시나리오 | 소요 시간 | 위협도 |
|---|---|---|
| Vercel이 "Security Tab" 추가 | 2-4개월 | 치명적 |
| GitHub Copilot에 보안 체크 통합 | 3-6개월 | 치명적 |
| Cursor/Windsurf가 빌트인 보안 추가 | 1-3개월 | 치명적 |
| Snyk이 "Easy Mode" UI 출시 | 1-2개월 | 높음 |
| Claude Code가 보안 체크 내장 | 이미 부분적으로 가능 | 높음 |

핵심 차별화 "쉬운 언어, 설치 없이 시작"은 **UX 레이어의 차별화**이지 **기술적 해자(Moat)가 아님.**

**Moat 평가:**
- 네트워크 효과: 없음
- 전환비용: 매우 낮음
- 데이터 우위: 잠재적이나 초기 없음
- 브랜드: 없음 (신규)
- 규모의 경제: 제한적

### 차별화 가능성: 2-3x (10x 아님)

"같은 것을 더 쉽게" 수준. **대안**: 바이브코딩 워크플로우에 네이티브 통합하여 코드 생성 시점에 실시간 개입하는 구조로 전환비용을 발생시켜야 함.

### 타이밍: 적절하나 윈도우 좁음 (12-18개월)

1. AI 코딩 도구 자체가 보안 체크를 내장할 것
2. 배포 플랫폼이 원클릭 보안을 제공할 것
3. 바이브코더가 성장하면 결국 개발자 도구로 이동

## 2. 실현 가능성 평가

### 기술적 실현 가능성: 높음 (조건부)

**출시 전 보안 스캔**: 충분히 가능. 오픈소스 도구로 상당 부분 커버.

**출시 후 실시간 모니터링**: 기술적 도전 존재.

| 기능 | 난이도 | 이유 |
|---|---|---|
| 브루트포스 감지 | 중 | 로그 분석 패턴 매칭 |
| 야간 접근 감지 | 중 | 시간대 기반 규칙 |
| 이상 위치 감지 | 중-고 | GeoIP + 이상탐지 모델 |
| 카드 테스트 감지 | 고 | 결제 패턴 분석, Stripe 연동 |
| 데이터 수집 감지 | 고 | 정확한 정의와 탐지 모델 |

**핵심 문제**: "설치 없이 시작"과 출시 후 모니터링(에이전트 설치 필수) 사이 괴리.
**대안**: Vercel/Netlify 로그 API 활용 "원클릭 연동"으로 타협.

### 유닛 이코노믹스 검토: 숫자가 빡빡함

**12개월 목표 역산:**
- 유료 1,200명 × ARPU $12.5 = MRR $15,000
- Effective CAC: $187-375
- LTV (이탈률 3%): $417
- **LTV/CAC: 1.1x - 2.2x** (건강 기준 3:1 미달)

| 시나리오 | 월 이탈률 | LTV | LTV/CAC |
|---|---|---|---|
| 낙관 | 3% | $417 | 1.1-2.2x |
| 현실 | 5% | $250 | 0.7-1.3x |
| 비관 | 7% | $179 | 0.5-1.0x |

**핵심 경고**: 바이브코더는 프로젝트 단위로 움직임 → 구조적으로 높은 이탈률.

**대안:**
- 연간 결제 할인 공격적으로 밀기 (40% 할인)
- "포트폴리오" 개념: 여러 앱을 하나 계정에서 관리
- "Secured by Guardrail" 배지로 바이럴 루프 구축하여 CAC 극소화

## 3. 리스크 평가

### 시장 리스크: 🟡 중-고 (확률 35%, 영향 치명적)

핵심 불확실성:
1. 바이브코더의 앱이 실사용자를 가지는 비율은?
2. AI 도구가 보안을 자동 해결하는 속도
3. 바이브코더가 보안 사고를 경험하는 빈도

### 실행 리스크: 🟢 낮-중 (확률 20%, 영향 중간)

- 오탐(False Positive): 신뢰 급락
- 미탐(False Negative): **법적 책임** 문제

### 경쟁 리스크: 🔴 높음 (확률 60%, 영향 치명적)

**가장 심각한 리스크.** 18개월 내 최소 하나의 대형 플레이어가 유사 기능 제공할 확률 **85% 이상**.

**대안:**
- 바이브코딩 보안 위협 데이터베이스 축적 (시간 = 우위)
- 커뮤니티/교육 생태계로 브랜드 해자
- 전략적 인수를 Exit 전략으로 명시적 설정

### 규제 리스크: 🟡 중간 (확률 25%, 영향 높음)

- "안전합니다" 표시의 법적 책임
- 사용자 코드 스캔 시 민감 정보 접근 이슈

## 4. PRD 약점 & 개선 제안

### 약점 1: 해자(Moat) 부재 — 가장 치명적

**문제**: UX 레이어 개선은 2-4주면 복사 가능. Snyk이 "쉬운 모드" 토글 하나 추가하면 끝.

**해결:**
1. "바이브코딩 보안 위협 DB" 구축 → 복사 불가 데이터 자산
2. IDE/AI 도구 네이티브 플러그인 → 전환비용 발생
3. 커뮤니티 해자 → 가장 강력하고 복사 어려움

### 약점 2: 유닛 이코노믹스의 구조적 불안정

**문제**: LTV/CAC 최선 2.2x, 건강 3:1 미달. 바이브코더 프로젝트 단위 행동 → 높은 이탈률.

**해결:**
1. $19 → $29 가격 인상 + 가치 추가
2. "앱 수" 기반 과금으로 자연 증가 구조
3. 연간 결제 50% 할인으로 이탈 구조적 완화
4. B2B2C (부트캠프 번들)로 CAC $5 이하

### 약점 3: "마찰 제로" 약속과 모니터링 현실 괴리

**문제**: 출시 후 모니터링은 에이전트 설치 필수. "마찰 제로"와 충돌.

**해결:**
1. Vercel/Netlify API 활용 "OAuth 한 번"으로 모니터링 시작
2. 1단계: 외부 관찰 항목만 → 2단계: 심층 모니터링
3. "설치 없이" → "3분 안에 연결"로 메시지 수정

## 5. 핵심 가설 & Kill Criteria

### 가설 1: "바이브코더는 보안에 돈을 지불할 의사가 있다"

- **Kill**: 6개월 시점 유료 전환율 < 2% → 즉시 피벗 검토
- **Kill**: 유료 사용자 첫 달 이탈률 > 30% → 가치 전달 실패

### 가설 2: "바이브코더의 앱은 보안이 필요한 수준의 실사용자를 가진다"

- **Kill**: 무료 스캔 중 실사용자 100명+ 앱 비율 < 5% → 시장 재평가
- **Kill**: 결제/개인정보 처리 앱 비율 < 15% → 보안 필요성 약함

### 가설 3: "대형 플랫폼이 이 니치를 직접 공략하기까지 18개월 이상"

- **Kill**: Vercel/Netlify가 무료 보안 대시보드 출시 → 6개월 내 차별화 재정의 못하면 중단
- **Kill**: 위 이벤트 시 유료 500명 미만이면 경쟁 불가

### 최종 판단: Conditional Go

**조건:**
1. 해자 전략 추가 (데이터베이스, IDE 통합, 커뮤니티 중 최소 1개)
2. 가격 구조 재검토 (ARPU $18-20+)
3. Kill criteria 팀 공유 및 매주 추적
4. 3개월 체크포인트 의무화
5. Exit 전략 명시 (전략적 인수 포함)
